Спрайт - базовая единица для отображения
Все спрайты должны жить в своиз пулах, т.к. события изменения размера, отрисовки и пересчета вызываются для пулов.
Также пулы следят за уничтожением своих объектов.

Пул спрайтов может быть как пулом однотипных переиспользуемых объектов, таким, как BaseSpritePool кораблей,
либо пулом спрайтов, которые не нуждаются в цикличном уничтожении/переиспользовании - это DefaultSpritePool.

На каждом базовом экране есть свой пул пулов, в котором по умолчанию содержится только DefaultSpritePool,
остальные должны быть добавлены соответствующей командой.

По умолчанию для всех пулов спрайты отрисовываются в порядке добавления СпрайтПула в пул спрайтпулов,
за исключением DefaultSpritePool. Для DefaultSpritePool доступно свойство DefaultPoolSpriteLayout,
которое отвечает за последовательность отрисовки спрайта - если оно установлено в BACKGROUND,
такие спрайты отображаются на экране первыми (в порядке добавления в DefaultPool). Это нужно,
например, для отрисовки фона. Если свойство установлено в FOREGROUND - такие спрайты будут
отрисованы после отрисовки всех остальных спрайтпулов. Это нужно для отображения счета
и элементов управления.

ВАЖНО! Если спрайт не добавлен в пул, для него не будет вызвано событие WorldResize, и
вероятно, что на момент его добавления он будет жить в своих координатах.

Спрайт, имплементирующий интерфейс InputListener можно подписать на события, происходящие
с тачпадом, мышкой или клавиатурой. По умолчанию в пакете есть кнопка класса BaseButton, имплементирующая
данный интерфейсю
